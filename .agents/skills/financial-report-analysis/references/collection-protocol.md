# 近期优先的信息采集协议

## Stage A：只收集，不分析

先按 `research-package.md` 建立独立来源目录、来源索引、提取数据表、主张映射和 `financial_collection_status`，不得评分。强制收集最近正式财报及版本、关键附注、分部、最近电话会和问答、90天内业绩预告/券商前瞻/预测调整、监管与会计事件、至少两家权威媒体、3—5家同行及重大股价异动原因。

必须使用 EasyAnySearch 分别执行中英文查询，覆盖 `earnings call/transcript/Q&A`、`earnings preview/estimate revision`、媒体/渠道、监管/诉讼、行业/同行和股价异动。搜索结果只是线索；尽量回到公司、交易所、SEC、监管、券商原文或第二独立来源核验。

## 新鲜度

- current：事件或最新报告在30天内。
- recent：31—90天。
- historical：超过90天但属于五年基线。
- stale：已被更正、重述、替代或不足以支持当前状态。

近期不等于权威。近期弱媒体不能覆盖旧但仍有效的正式财报。每个已覆盖项目必须有提取事实或数据，只有URL、标题或搜索摘要视为缺失。

## 门禁

最新正式财报、正确版本、币种/单位或核心三表冲突任一缺失，`blocked`且只能输出采集报告。电话会、近期预测/监管/媒体/同行缺失，通常为`partial`且只能输出阶段性判断。全部满足或明确不适用才为`pass`，允许完整分析。

每条预测记录原预测、新预测、机构、日期、口径、依据和未来正式验证事件。历史趋势与近期信号冲突时，单列传导机制，禁止用五年平均掩盖拐点。
