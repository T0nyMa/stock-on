# 日报当日交易评价设计

## 目标

将 2026-07-14 日报的技术信息重组为“自然语言先给出当日交易判断、技术指标提供可追溯证据”的统一版式。全部 22 只追踪股票均须有当日评价和下一交易日观察条件。

## 覆盖与边界

- 保持单一七章日报：`tracking/daily/positions/2026-07-14.md`；HTML 只能由该 Markdown 派生。
- 核心 5 只股票使用完整交易评价卡；其余 17 只使用统一指标矩阵和逐只两句评价。
- 只使用现有 `daily_snapshot.json`、`report_context.json` 和已登记港股行情/K 线数据。没有登记量比、日内 OHLC 或均量时，必须写 `unavailable`，不得以换手率推定“放量”。
- 不改动交易记录、持仓、行情或指标快照。

## 核心股交易评价卡

在原有消息、研究和操作判断之外，固定加入以下七项：

| 维度 | 指标证据 | 写作输出 |
|---|---|---|
| 当日表现 | 收盘、涨跌幅 | 强于/弱于/跟随市场 |
| 日内结构 | OHLC、振幅、收盘位置 | 高开高走、冲高回落、探底回升、横盘或 `unavailable` |
| 量能与资金 | 量比/均量、换手、CMF、OBV | 放量/缩量结论，或量能 `unavailable` |
| 趋势位置 | MA5/10/20、MACD、ADX | 延续、回调、震荡或转弱/转强候选 |
| 动量与过热 | RSI6/12、布林带位置 | 强势、偏热、修复或动能不足 |
| 波动与风险 | ATR/NATR、振幅、布林带宽度 | 正常、高波动或异常分歧 |
| 关键位与行动 | 支撑、阻力、观察区、失效、目标、RR | 明日确认和失效条件 |

卡片顶部以粗体自然语言写一段“当日评价”，再以“明日观察”给出可验证条件；不能把反弹候选写成趋势反转。

## 观察股矩阵与评价

三组观察股表保留每只一行，字段为：收盘/涨跌、趋势、MA5/10/20、MACD、RSI6/12、ADX、MFI/CMF、关键位和结论。每只股票在矩阵之后紧跟两句：

- `当日评价`：当日价格表现、可用的量价/资金证据，以及在趋势中的含义。
- `明日观察`：结构确认条件和失效条件。

当日 OHLC 或量比缺失时，不使用“探底回升”“冲高回落”“放量上涨”“缩量回调”等精确日内/量能措辞，改为说明该维度 `unavailable`，仍可依据收盘、均线、CMF 等给出有限结论。

## 验证

- 七章完整，核心股 5 张交易评价卡齐全。
- 22 只股票各自在核心区或观察区覆盖；17 只观察股均有“当日评价”和“明日观察”。
- 所有“放量/缩量”表述均有登记量比或均量证据；否则显式为 `unavailable`。
- HTML 从更新后的 Markdown 重新生成，包含全部章节和内容；只提交报告 Markdown 与 HTML。
