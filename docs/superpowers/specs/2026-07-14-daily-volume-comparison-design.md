# 日报量价与指标变化设计

## 目标

让日报的短线交易评价同时展示当日量价状态、相对昨天的变化和相对近期常态的变化，使“放量上涨”“缩量回调”“趋势增强”等结论可追溯，而非仅展示静态技术指标。

## 研究依据

- Fidelity 将技术分析拆为趋势、动量、成交量、波动和支撑阻力；MACD、RSI、ADX、MFI/CMF 是不同类别的证据，不能互相替代。
- Fidelity 对 MACD 的说明指出震荡区间容易产生反复交叉；因此日报必须以趋势状态和关键位过滤单一 MACD 信号。
- CMT Association 的量能研究将成交量用作价格行动的确认、吸筹/派发判断及风险管理输入，而不是单独买卖信号。

## 现有数据能力与缺口

- SQLite quote 已登记 A 股当日 `volume`、`amount`、`turnover_rate`；日线 bars 已登记 OHLCV 与成交额。
- 日线数据层的 `volume_ratio` 定义为“当日成交量 / 前 5 日平均成交量（不含当日）”，适合日度放量判断，不与盘中分时量比混用。
- 当前 `daily_snapshot.json` 只序列化换手率，遗漏成交量、成交额、量比、昨量/昨成交额、量能变化和 OHLC，致使日报不能严格描述日内结构或量价关系。

## 目标快照接口

为每只 A 股在 `daily_snapshot.json.a_stocks[]` 增加可空字段：

```json
{
  "prev_close": 0,
  "open": 0,
  "high": 0,
  "low": 0,
  "volume": 0,
  "amount": 0,
  "volume_ratio_5d": 0,
  "volume_change_vs_prev": 0,
  "amount_change_vs_prev": 0,
  "turnover_change_vs_prev": null,
  "amplitude": 0,
  "close_position": 0,
  "macd_hist_change": 0,
  "rsi6_change": 0,
  "rsi12_change": 0,
  "adx_change": 0,
  "mfi14_change": 0,
  "cmf20_change": 0
}
```

数值缺失一律序列化为 JSON `null`；报告层显示为 `unavailable`，不得填零。成交量与成交额的昨日变化以 `(今日/昨日)-1` 表示；量比使用前 5 日均量基准；换手率昨日变化仅在两个同口径换手率均已登记时输出。

## 日报判定规则

日报将每只股票的“量能与资金”改为三层：

1. **绝对参与度**：成交量、成交额、换手率。
2. **比较变化**：成交量/成交额较昨日变化，换手率较昨日变化，量比（相对前 5 日均量）。
3. **量价解释**：仅在价格方向和量比均可用时分类。

| 分类 | 条件 |
|---|---|
| 放量上涨 | 涨幅 > 0 且量比 ≥ 1.2 |
| 强放量上涨 | 涨幅 > 0 且量比 ≥ 1.5 |
| 放量下跌 | 涨幅 < 0 且量比 ≥ 1.2 |
| 缩量回调 | 涨幅 < 0 且量比 ≤ 0.8 |
| 量能正常/未确认 | 0.8 < 量比 < 1.2，或缺少任何必要字段 |

“突破确认”还需要收盘突破已登记阻力或观察区上沿；“日内反转”还需要 OHLC、收盘位置和振幅证据。MACD、RSI、ADX、MFI、CMF 的报告解释同时呈现当前值与较前一交易日变化，但不把单日变化单独升级为交易指令。

## 报告版式

- 核心五只的七维评价卡中，日内结构补入 OHLC、振幅、收盘位置；量能与资金补入成交量/额、换手、量比和昨日变化；趋势、动量、资金指标补入日变化。
- 17 只观察股矩阵增加紧凑的“量能/变化”列；逐只自然语言评价使用标准量价分类或 `unavailable`。
- 所有操作仍由关键位、触发条件、失效条件和仓位共同决定，指标仅作为证据。

## 验证

- 为快照派生函数增加单元测试：含完整两日/六日数据时正确计算量比、昨日变化、振幅和收盘位置；数据不足时输出 `None`。
- 验证报告不在必要字段缺失时使用放量、缩量、反转等分类。
- 验证七章、22 只覆盖、核心五卡与 HTML 渲染完整；仅提交与本需求相关的数据代码、测试及报告文件。
