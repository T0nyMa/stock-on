# 日报价量结构设计

## 目标

在登记的 `daily-report` 工作流中，为全部持仓股和第三章固定核心股增加统一、可复核的价量结构分析。指标由确定性数据层计算，日报只消费登记结果并生成解释，避免生成阶段临时计算或形成多套口径。

## 范围

### 覆盖股票

每日构造以下股票集合并按代码去重：

1. `tracking/tracklist.json` 中 `has_position=true` 的全部股票；
2. 日报第三章固定核心股：工业富联、山东黄金、三花智控、兆易创新、阿里巴巴。

A/H两地上市股票按市场分别处理。只有具备同口径历史数据的市场才输出完整价量指标；不得把A股指标映射为H股结论。

### 报告位置

不增加独立章节或总表。每只适用股票的技术卡中增加一行：

`价量结构：{指标摘要}；{分类与解释}；{证据缺口}`

第二章持仓卡和第三章核心股卡复用同一份登记结果，不重复计算。

## 指标口径

目标交易日记为 `T`。所有日线计算只使用日期不晚于 `T` 的登记K线。

| 字段 | 定义 | 最低历史要求 |
|---|---|---:|
| `price_change_pct` | T日收盘涨跌幅 | 2个交易日或登记涨跌字段 |
| `intraday_volume_ratio` | 登记行情量比；数据源缺失时不以其他指标冒充 | 登记行情可用 |
| `volume_vs_ma5` | T日成交量 ÷ 最近5个交易日平均成交量（含T） | 5个交易日 |
| `volume_vs_ma20` | T日成交量 ÷ 最近20个交易日平均成交量（含T） | 20个交易日 |
| `recent20_vs_previous20` | 最近20日平均成交量 ÷ 此前20日平均成交量 | 40个交易日 |
| `up_down_volume_ratio_90d` | 最近90个自然日内上涨日平均成交量 ÷ 下跌日平均成交量 | 上涨日和下跌日均至少1日 |
| `mfi14` | 登记指标快照的MFI14 | 登记指标可用 |
| `cmf20` | 登记指标快照的CMF20 | 登记指标可用 |
| `obv_20d_direction` | 登记OBV序列近20个交易日变化方向：`rising/falling/flat` | 21个交易日 |
| `turnover_rate` | 登记行情换手率 | 登记行情可用 |

除上述比率外不引入新的技术公式。数值保留两位小数。分母为0、字段非数值或历史不足时返回 `None`，渲染为 `unavailable`。

## 价量分类

分类以 `volume_vs_ma20` 为主，`volume_vs_ma5` 和 `intraday_volume_ratio` 只解释短期变化：

- `expanding`：`volume_vs_ma20 >= 1.20`
- `contracting`：`volume_vs_ma20 <= 0.80`
- `normal`：其余可用值
- `unavailable`：`volume_vs_ma20` 不可用

结合T日价格方向形成基础标签：

| 价格 | 量能 | 标签 |
|---|---|---|
| 上涨 | expanding | 放量上涨 |
| 上涨 | contracting | 缩量上涨 |
| 下跌 | expanding | 放量下跌 |
| 下跌 | contracting | 缩量下跌 |
| 平盘或 normal | 任意 | 正常量能/量价中性 |

当 `volume_vs_ma5 >= 1.20` 且 `recent20_vs_previous20 <= 0.80` 时，优先补充“局部放量但中期缩量”，但不覆盖基础价格方向。

## 资金验证与解释边界

- `CMF20 > 0`、OBV近20日上升可作为正向验证；`CMF20 < 0`、OBV下降可作为负向验证。
- MFI仅描述资金强弱位置，不单独决定放量或缩量。
- 上涨放量不自动等于有效突破，仍需价格越过登记阻力与趋势强度确认。
- 下跌缩量不自动等于止跌，仍需不再创新低、结构修复或资金改善。
- 下跌放量且CMF为负时，解释为负向量价结构；不得弱化为普通波动。
- 事实指标、技术解释和持仓动作保持分层；价量模块不直接生成交易指令。

## 数据流

1. `fetch-data` 将目标日行情及历史K线写入SQLite。
2. `tech-indicators` 写入MFI、CMF、OBV等登记指标快照。
3. `src/daily_metrics.py` 从登记K线和指标输入派生价量字段及分类。
4. `run_quant_analysis.py`/报告上下文生成流程把结果写入每只股票的 `artifact.report_context`。
5. `daily-report` 读取同一结果，在第二章持仓卡及第三章核心股卡渲染“价量结构”行。
6. HTML由登记Markdown机械生成，不重新计算指标。

## 代码与登记变更

- 扩展 `src/daily_metrics.py`，输出完整价量字段、分类和证据缺口。
- 扩展 `tests/test_daily_metrics.py`，固定计算和边界行为。
- 将价量结果接入当前生成 `data/report_context.json` 的路径；不得在Markdown生成器内重复公式。
- 更新 `references/templates/daily-report-v2.md`，为持仓卡和核心股卡加入价量结构占位。
- 更新 `.agents/skills/daily-report/SKILL.md`，将价量字段列为持仓股与核心股的必读证据。
- 如登记schema限制新字段，则以向后兼容的可选对象加入；旧上下文缺失时统一渲染 `unavailable`。

## 测试

### 单元测试

至少覆盖：

1. 放量上涨；
2. 放量下跌且CMF为负；
3. 缩量上涨；
4. 当日相对MA5放量、但近20日相对此前20日缩量；
5. 只有1、5、20、39和40个交易日的历史边界；
6. 成交量为0、分母为0、字段为`None`或非数值；
7. 上涨日或下跌日样本缺失；
8. OBV上升、下降、持平和不可用；
9. H股历史缺失时不得消费A股结果。

### 集成验证

- `report_context` 中持仓股和核心股均存在价量对象或明确的 `unavailable`。
- 日报第二章、第三章的适用股票卡均包含“价量结构”。
- 同一股票在两个章节中的数值一致。
- 日报保持七章结构，观察股章节不强制展开完整价量行。
- Markdown和HTML均保留缺失字段，不出现猜测值。

## 完成标准

1. 价量指标由单一确定性函数计算并通过测试；
2. 持仓股和核心股在日报卡片内稳定显示；
3. 分类、资金验证和解释边界符合本设计；
4. `unavailable`、历史不足和跨市场隔离均有测试；
5. 日报工作流、Skill、模板和数据上下文口径一致；
6. 完整日报生成、发布及线上验证通过。
