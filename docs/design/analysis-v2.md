# Analysis V2 Design Doc

## 两个项目的分析哲学差异

```
stock-analysis:  数据质量 > 叙事完整性。缺数据就标 gap，不编。
stock-on:        策略信号 > 操作决策。有持仓要管，有浮亏要处理。
```

一个追求证据可靠性，一个追求决策可操作性。优化的方向是**把前者的严谨性引入后者的决策框架**。

---

## 现状诊断

### 本项目分析体系的优势

| 能力 | 说明 |
|------|------|
| 15策略共识决策 | 多策略从不同角度评分，decision agent 加权汇总，消除单一策略噪音 |
| 市场状态路由 | regime detection → 按趋势/震荡/空头推荐不同策略组合 |
| A-H双市场对比 | 同公司不同市场定价差异，溢价分析，港股PE对比 |
| 持仓闭环跟踪 | position.json 记录成本/股数/浮亏 → 日报更新 → 操作建议追踪 |
| 消息面整合 | AnySearch → 提取关键公告/业绩/新闻 → 融入技术分析 |
| 七章日报 | 宏观→持仓→深度→A-H→概览→策略→操作，结构完整 |

### 三个核心缺陷

**缺陷1: 没有证据质量概念**

Claude 读 `indicators.json` 做分析时，不知道这个数据是刚从腾讯拉到的实时数据，还是三天前的缓存。数据源挂了的时候，可能拿着不完整的数据在做分析。

stock-analysis 的做法：每个证据模块标注 available/missing，report 里明确写"北向资金数据不可用"。

**缺陷2: 分析只有单一视角**

现在所有分析都是 Claude 的"通用视角"。没有巴菲特式的"这家公司值不值得持有10年"视角，也没有西蒙斯式的"统计上这个模式胜率多少"视角。

stock-analysis 的做法：同一份 M1-M6 证据，16 种投资人透镜重新加权解读，可以委员会模式（6个透镜综合）或对抗模式（2个透镜辩论）。

**缺陷3: 报告生成依赖 LLM，不可审计**

每天的报告是 Claude 当场写的。同样的数据，两次"日报"命令可能产生表述不同、结论不同的报告。没有中间审计层。

stock-analysis 的做法：先产出 JSON Evidence Pack（可 diff 的数据层），再基于证据生成报告。报告结构和证据之间有清晰的映射关系。

## 改进方案：三层分析框架

```
Layer 1: 证据层 (Evidence)        — 数据+质量标记
Layer 2: 视角层 (Perspectives)    — 多策略+多透镜
Layer 3: 决策层 (Decision)        — 共识+操作+审计
```

### Layer 1: 证据层

在现有技术指标基础上，新增证据质量标记：

```python
# 在 indicators.json 中增加元数据
{
  "data": { ... },  # 现有的 MA/MACD/RSI 等
  "_evidence": {
    "quality_score": 85,        # 0-100，参考 stock-analysis 的模块评分
    "source": "tencent",        # 数据来源
    "fetched_at": "...",        # 拉取时间
    "gaps": [],                 # 缺失的数据类型
    "stale": false              # 是否可能过期
  }
}
```

**日报报告中新增证据质量声明**（报告第一章前插入）：

```
> 证据质量: 85/100 | 数据源: 腾讯+东财 | 缺失: 北向资金 | 拉取: 15:32
```

这个声明的价值：Claude 知道什么数据是可靠的，什么需要保守判断。

### Layer 2: 视角层

在现有15策略基础上，增加三层视角：

#### 2a. 策略层（已有，优化）

现状是15个策略，改进点：
- 每个策略输出增加**置信度理由**（不仅给评分，还要说"为什么这次判断可信/不可信"）
- 增加**策略历史胜率追踪**：记录每次信号后N天的实际走势，计算该策略在特定市场状态下的历史准确率

#### 2b. 投资人透镜层（新增，学习 stock-analysis）

选定 5 个与中国市场最相关的透镜：

| 透镜 | 核心问题 | 证据偏好 |
|------|---------|---------|
| **巴菲特** | 这公司值不值得持有10年？护城河在哪？ | M2(行业)+M5(质量)+基本面 |
| **格雷厄姆** | 安全边际够不够？下跌保护充分吗？ | M4(风险)+估值+资产负债表 |
| **西蒙斯** | 统计上这个模式胜率多少？信号可靠性？ | M3(动量)+量价+历史统计 |
| **段永平** | 商业模式好吗？企业文化靠谱吗？ | M5(质量)+管理层+竞争力 |
| **索罗斯** | 市场的共识是什么？共识错在哪？ | M1(宏观)+M2(资金流)+反身性 |

透镜不替代策略，而是在策略评分之上增加一层"如果XXX看这个问题，他会关注什么"的叙事框架。

**日报中的应用**：

```
## 三、兆易创新（603986）深度分析

### 多视角评估

| 视角 | 核心判断 | 关注点 |
|------|---------|--------|
| 策略共识 | 58分/hold | 底部地量+MACD底背驰，但PE162估值压制 |
| 巴菲特视角 | ❌ 不投 | 芯片周期股，护城河不够深，10年确定性低 |
| 西蒙斯视角 | ⚠️ 谨慎 | 地量+RSI超卖统计上看反弹概率65%，但样本量不足 |
| 段永平视角 | ⚠️ 观望 | 存储/MCU是好赛道，但管理层+企业文化需要验证 |
```

#### 2c. 情绪量化层（新增）

目前消息面是 Claude 读完新闻写主观总结。改为先量化再解读：

```
新闻分析:
  来源: 财联社/东财/富途
  相关度评分: 9/10
  情感评分: +0.7 (正面)
  紧急性: 高 (业绩预告)
  摘要: H1净利润同比+93%-101%
```

参考 stock-analysis 的 `chinese_news_analysis` 结构。

### Layer 3: 决策层

#### 3a. 决策仪表盘（已有 decision agent，优化）

增加**情景概率**替代单一判断：

```
操作建议: 工业富联（601138）
├── 情景A (概率45%): 业绩预告后持续反弹 → 目标75-78 → 减仓1000股
├── 情景B (概率35%): 68-72区间震荡 → 持有不动 → 等待Q3指引
├── 情景C (概率20%): 大盘风险+利好出尽 → 跌破65 → 减仓500股
└── 建议: 持有，72以上分批减仓，止损64
```

现在的日报已经有情景判断，但概率是主观估计。加入量化支持会更好。

#### 3b. 决策审计追踪

在 `position.json` 中新增决策日志：

```json
{
  "decision_log": [
    {
      "date": "2026-07-09",
      "action": "hold",
      "price": 69.53,
      "reason": "业绩预告超预期验证底部判断，反弹中不加仓，等72以上减仓",
      "evidence_quality": 85,
      "strategies_consensus": {"buy": 2, "hold": 3, "sell": 0}
    }
  ]
}
```

这样可以回溯每次决策时的证据状态和策略信号。

## 报告结构优化

### 现状: 七章日报

```
1. 宏观背景  2. 持仓追踪  3. 核心个股深度  4. A-H对比
5. 其余观察股  6. 策略信号  7. 操作清单
```

### 建议: 保持七章结构，每章增强

| 章节 | 现有 | 增强 |
|------|------|------|
| 宏观背景 | 四大指数+金价+板块TOP5 | +北向资金(新增数据) +证据质量声明 |
| 持仓追踪 | 成本/现价/浮亏 | +决策日志链接 +情景概率 |
| 核心个股深度 | 数据表→消息面→技术面→共振 | +多视角表(策略+透镜) +情绪量化 |
| A-H对比 | 溢价+PE表 | +港股透镜视角(外资怎么看) |
| 其余观察股 | 表格式 | +异常检测标记(量异常/价异常) |
| 策略信号 | 评分+依据 | +策略历史胜率 +置信度标注 |
| 操作清单 | 每只一句话 | +情景概率 +决策审计记录 |

## 与 stock-analysis 的对比

| 维度 | stock-analysis | stock-on V2 |
|------|---------------|-------------|
| 分析哲学 | 证据驱动，缺数据标 gap | 决策驱动，缺数据降置信度 |
| 核心框架 | M1-M6 六模块 | 15策略 + 5透镜 + 情绪量化 |
| 多视角 | 16透镜，委员会/对抗模式 | 5透镜，嵌入日报深度分析 |
| 质量评分 | 100分制，模块级 | 100分制，个股级 |
| 情绪量化 | 新闻情感分+社区讨论分 | 新闻情感分+紧急性+相关性 |
| 报告 | 确定性模板，LLM可消费 | Claude生成，结构固定 |
| 可审计性 | JSON Evidence Pack 可 diff | 决策日志 + evidence 元数据 |
| 操作建议 | 无 | 具体价位+股数+情景概率 |
| 持仓追踪 | 有 portfolio profile | 有 position.json+决策日志 |

## 实现计划

### Phase 1: 证据质量标记（1天）

- `indicators.json` 增加 `_evidence` 字段
- 数据拉取时记录 source/fetched_at/gaps
- 日报开头增加证据质量声明行

### Phase 2: 5透镜定义（1天）

- 创建 `references/lenses/` 目录
- 定义 buffett/graham/simons/duan_yongping/soros 五透镜
- 每透镜：核心问题 + 证据权重 + 分析框架

### Phase 3: 情绪量化（1天）

- 消息搜索结果结构化（来源/相关度/情感/紧急性）
- 替代纯主观消息面解读

### Phase 4: 决策审计（0.5天）

- `position.json` 增加 `decision_log`
- 每次日报更新时追加一条记录

### Phase 5: 策略历史胜率（2天）

- 回溯历史 strategy_*.json 信号
- 计算每种策略在各类市场状态下的实际胜率
- 在策略信号章节展示

- [ ] 每份日报开头有证据质量声明

- [ ] 操作清单含情景概率
- [ ] position.json 有完整决策日志
- [ ] 策略信号含置信度理由
